

# Визуальный Link Prediction на графах знаний с использованием LVLM

## Описание проекта

В рамках проекта была реализована система **визуального Link Prediction** на графе FB15k-237 с использованием Large Vision-Language Model (LVLM). 

Основная идея заключается в том, чтобы вместо традиционного текстового представления графа подавать модели **изображение подграфа**, сгенерированное с помощью Graphviz. Модель (LLaVA-1.5-7B) обучается предсказывать недостающую сущность по визуализированному окружению головы.

Для повышения качества обучения применён подход **Curriculum Learning**: сначала модель обучается понимать структуру графа (Stage 1), а затем решает целевую задачу link prediction (Stage 2).


## Архитектура пайплайна

**1. Загрузка и хранение графа.** Датасет FB15k-237 загружается в Neo4j с сохранением метаданных (человекочитаемые имена, описания сущностей), пакетной дедупликацией троек и явным разделением на train/valid/test сплиты через свойство рёбер. Созданы индексы для ускорения выборок.

**2. Извлечение head-centric подграфов.** Для каждой тройки строится подграф исключительно вокруг head-сущности. Реализована кумулятивная фильтрация по сплитам: train видит только train-рёбра, valid — train+valid, test — все.

**3. Визуализация для LVLM.** Подграфы рендерятся GraphViz непосредственно в целевом разрешении модели (336×336 px) Head выделен цветом (синий), все соседи — нейтральные. К графу добавляется пунктирный узел-заглушка `[?]` с подписью целевого отношения — модель видит, какую связь нужно предсказать, но не видит ответа. 

**4. Генерация датасета (Curriculum Learning).** *Stage 1 (Visual Graph Comprehension):* self-supervised задачи — подсчёт узлов, рёбер, определение степеней вершин, перечисление триплетов. Все ответы соответствуют тому, что визуально присутствует на изображении: counts включают узел `[?]` и пунктирное ребро к нему. *Stage 2 (Link Prediction QA):* супервизированные примеры, где tail фигурирует исключительно в поле `answer`. Промпты ссылаются на `[?]` как на целевой узел, описывают head-сущность и тип связи.

**6. Обучение и оценка.** QLoRA fine-tuning (4-bit NF4, bf16, gradient checkpointing) через стандартный HuggingFace Trainer с early stopping. Оценка — batched beam-search генерация с метриками MRR, Hits@1/3/10, Found Rate.
